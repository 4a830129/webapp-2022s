[實作問題 4]：本地端(Client-side)『表單(Forms)』應用。

  • 參考作業：第5周 課堂作業 1

基本題：

  • 這是一個與作業『完全』相同的考題。
  • 利用 Javascript 對輸入欄位做簡單的驗證，Javascript 程式必須是一個外部的程式檔 main.js。
  • 當 Click 按鍵被點擊時，檢查表單(form)中的文字輸入框(text input)是否為空的。
  • 文字框如果為空的，輸出 『同學你好，你好像忘了輸入名字哦。』 訊息。
  • 文字框若有資料(xxx)，輸出對應的 『xxx 同學你好，歡迎加入我們的 HTML/CSS 學習。』 訊息。

加分題：

  • 觀察在文字輸入框中直接按『enter』鍵盤的反應。
  • 他並無法像當 Click 按鍵被點擊時，進行文字對話框資料檢查及對應輸出。
  • 這是因為我們並沒有處理 <input type="text".....> 中的『change』事件。
  • 請在網路中找尋解決方案，將『change』事件的處理程式加入，使其擁有像Click 按鍵相同的反應。

[實作問題 5]：簡易動態『表格(Tables)』頁面設計。

  • 參考作業：第5周 課堂作業 2

基本題：

  • 這是一個與作業『完全』相同的考題。
  • 當 Update 按鍵被點擊時，以隨機亂數的方式改變『120秒時』的『爐溫』。
  • 當 Add Record 按鍵被點擊時，以隨機亂數的方式增加一筆新資料： 資料格式：<隨機亂數> <隨機亂數> <隨機亂數> "隨機資料" 參考範例：11 203 178 隨機資料
  • Update 按鍵點擊更新的資料，須以『紅色』的方式呈現。
  • Add Record 按鍵點擊產生的資料，須以『深色背景』的方式呈現

加分題：

  • 當資料量到達『10筆』時，取消 Add Record 按鍵的『產生隨機資料』功能。
  • 也就是點擊 Add Record 按鍵，將不再增加新的資料，並在底部輸出『資料緩衝已滿!』的訊息。

注意事項： 基本題與作業100%相同，加分題難度適中，考驗你對JavaScript邏輯判斷的使用能力。

[實作問題 6]：JavaScript 應用 - 簡易乘法表。

  • 參考作業：第7周 課堂練習 5

進階題：

  • 這是一課堂練習的『進化』考題。
  • 增加成『9個』數字按鍵(1,2,3,...,9)。
  • 當任何一個數字按鍵被點擊時，對應輸出該數字的乘法表。
  • 9個按鍵必須固定在適當的位置，且要以置中『九宮格』的形式呈現。
  • 你『必須』為你的『按鍵』及『乘法表』套上顯眼的樣式(style)，『僅』功能完成正確的答案只有80%的成績。
  • CSS 及 Javascript 程式語必須是獨立的外部檔案。

[實作問題 8]：使用 JavaScript 依參數 n，繪製『n-層』等腰三角形。

  • 參考作業：第7週 課堂練習 9 + 第5週 課堂作業 1

進階題：

  • 這是一課堂練習的『進化』考題。
  • 參考輸出結果，設計簡易的使用者操作介面。
  • 輸入的整數必須介於 1 ~ 10 之間，超出範圍只要在底下印出『輸入整數超出範圍!』訊息。
  • 當 Draw 按鍵被點擊時，擷取表單中符合範圍的整數n，繪製出對應的『n-層』等腰三角形。
  • 表單中的所有元素，必須在瀏覽器中呈現置中的位置，否則將會被扣20%的成績。
